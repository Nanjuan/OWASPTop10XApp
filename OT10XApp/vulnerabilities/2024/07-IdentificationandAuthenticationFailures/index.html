What are Identification and Authentication Failures?

Identification and authentication failures occur when applications incorrectly implement authentication mechanisms, allowing attackers to assume users' identities. This includes weak password policies, session management flaws, and insecure authentication implementations.

Key Concepts:

  • Weak Passwords: Easily guessable or common passwords
  • Session Management: Insecure session handling and token management
  • Brute Force Attacks: Automated password guessing attempts
  • Credential Stuffing: Using leaked credentials from other breaches

Attack Scenarios

Scenario 1: Weak Password Policy

Beginner

Exploit weak password policies and common password patterns to gain unauthorized access.

Vulnerable Component:

Weak password validation and common password acceptance

Attack Steps:

  1. Identify weak password requirements
  2. Use common password lists
  3. Attempt brute force attacks
  4. Exploit predictable password patterns

Scenario 3: Credential Stuffing

Advanced

Use leaked credentials from data breaches to gain unauthorized access to accounts.

Vulnerable Component:

Reused passwords and lack of breach monitoring

Attack Steps:

  1. Obtain leaked credential databases
  2. Test credentials against target application
  3. Automate login attempts

Prevention Strategies

Strong Authentication

  • Implement strong password policies
  • Use multi-factor authentication (MFA)
  • Enforce password complexity requirements
  • Implement account lockout mechanisms

Session Management

  • Use secure session tokens
  • Implement proper session timeout
  • Secure session storage
  • Implement session invalidation

Security Monitoring

  • Monitor for suspicious login attempts
  • Implement breach monitoring
  • Use CAPTCHA for repeated failures
  • Regular security audits

Code Examples

Vulnerable Implementation

// WARNING: Weak password validation
function validatePassword(password) {
    return password.length >= 4; // Too weak!
}

// Insecure session token generation
function generateSessionToken() {
    return Math.random().toString(36); // Predictable!
}

// Plain text password storage
function storeUser(username, password) {
    users[username] = password; // Never store plain text!
}

// No rate limiting on login attempts
app.post('/login', (req, res) => {
    const { username, password } = req.body;
    if (users[username] === password) {
        res.json({ success: true });
    }
});

// Weak session management
app.get('/profile', (req, res) => {
    const sessionId = req.cookies.sessionId;
    const user = sessions[sessionId]; // No validation!
    res.json(user);
}

Secure Implementation

// Strong password validation
function validatePassword(password) {
    const minLength = 12;
    const hasUpperCase = /[A-Z]/.test(password);
    const hasLowerCase = /[a-z]/.test(password);
    const hasNumbers = /\d/.test(password);
    const hasSpecialChar = /[!@#$%^&*]/.test(password);
    
    return password.length >= minLength && 
           hasUpperCase && hasLowerCase && 
           hasNumbers && hasSpecialChar;
}

// Secure session token generation
function generateSessionToken() {
    return crypto.randomBytes(32).toString('hex');
}

// Secure password hashing
async function storeUser(username, password) {
    const saltRounds = 12;
    const hashedPassword = await bcrypt.hash(password, saltRounds);
    users[username] = hashedPassword;
}

// Rate limiting and secure authentication
const rateLimit = require('express-rate-limit');
const loginLimiter = rateLimit({
    windowMs: 15 * 60 * 1000, // 15 minutes
    max: 5 // limit each IP to 5 requests per windowMs
});

app.post('/login', loginLimiter, async (req, res) => {
    const { username, password } = req.body;
    const user = await getUser(username);
    
    if (user && await bcrypt.compare(password, user.password)) {
        const sessionToken = generateSessionToken();
        await storeSession(sessionToken, user.id);
        res.cookie('sessionId', sessionToken, { 
            httpOnly: true, 
            secure: true, 
            sameSite: 'strict' 
        });
        res.json({ success: true });
    } else {
        res.status(401).json({ error: 'Invalid credentials' });
    }
});

// Secure session validation
app.get('/profile', async (req, res) => {
    const sessionToken = req.cookies.sessionId;
    const userId = await validateSession(sessionToken);
    
    if (!userId) {
        return res.status(401).json({ error: 'Invalid session' });
    }
    
    const user = await getUserById(userId);
    res.json(user);
}
